# Internationalization (i18n) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add language switching to Meridian using i18next, with ~164 UI strings extracted from 32 components into per-language JSON files, community-submittable translations, and user-installable custom locales.

**Architecture:** All UI strings move into `src/renderer/src/i18n/locales/en.json` keyed as `"area.key"`. `react-i18next`'s `useTranslation()` hook replaces hardcoded strings. A `language` field in `useSettingsStore` persists the choice; `i18n.changeLanguage()` switches instantly. User-installed locales are loaded from `{userData}/locales/*.json` via a new IPC handler.

**Tech Stack:** i18next 23.x, react-i18next 14.x, Electron IPC, Zustand (existing store), TypeScript.

- [ ] **Step 3: Create `i18n/index.ts`**

Create `meridian/src/renderer/src/i18n/index.ts`:

```ts
import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'

export async function initI18n(language: string): Promise<void> {
  // Bundled locales — loaded dynamically so new ones don't require code changes
  const bundled: Record<string, object> = {}
  const modules = import.meta.glob('./locales/*.json', { eager: true }) as Record<string, { default: object }>
  for (const [path, mod] of Object.entries(modules)) {
    const lang = path.replace('./locales/', '').replace('.json', '')
    bundled[lang] = mod.default
  }

  // User-installed locales from {userData}/locales/
  const userLocales = await loadUserLocales()

  const resources: Record<string, { translation: object }> = {}
  for (const [lang, msgs] of Object.entries({ ...bundled, ...userLocales })) {
    resources[lang] = { translation: msgs }
  }

  if (i18n.isInitialized) {
    await i18n.changeLanguage(language)
    return
  }

  await i18n.use(initReactI18next).init({
    resources,
    lng: language,
    fallbackLng: 'en',
    interpolation: { escapeValue: false },
  })
}

async function loadUserLocales(): Promise<Record<string, object>> {
  try {
    const files: { lang: string; content: object }[] = await (window.vault as any).listUserLocales?.() ?? []
    const result: Record<string, object> = {}
    for (const { lang, content } of files) result[lang] = content
    return result
  } catch {
    return {}
  }
}

export { i18n }
```

ipcMain.handle(IPC.LOCALE_LIST_USER, async () => {
  const { existsSync, readdirSync, readFileSync } = await import('fs')
  const { join } = await import('path')
  const { app } = await import('electron')

  const dir = join(app.getPath('userData'), 'locales')
  if (!existsSync(dir)) return []

  const results: { lang: string; content: object }[] = []
  for (const file of readdirSync(dir).filter((f) => f.endsWith('.json'))) {

    } catch {
      // skip malformed files
    }
  }
  return results
})

```markdown
# Contributing a Translation

Thank you for translating Meridian! Here's how to add a new language.

## Steps

1. Fork this repository
2. Copy `src/renderer/src/i18n/locales/en.json` to `src/renderer/src/i18n/locales/XX.json`
   - Replace `XX` with the [ISO 639-1 code](https://en.wikipedia.org/wiki/List_of_ISO_639-1_codes) for your language
   - Examples: `fr.json` (French), `de.json` (German), `ja.json` (Japanese), `zh.json` (Chinese)
3. Translate all the **values** in the JSON file. **Do not change the keys.**
4. Open a Pull Request with the title: `i18n: add [Language name] translation`

## What to translate

Translate everything **except:**
- Keys (left side of each `:`  )
- Keyboard shortcuts: `⌘K`, `⌘S`, `⌘N`, etc.
- File extensions: `.md`, `.canvas`
- Brand names: `Meridian`, `GitHub`, `Git`, `Markdown`
- URLs

## Placeholders

Some strings contain placeholders like `{{count}}` or `{{name}}`. Keep these exactly as-is — they get replaced with real values at runtime:

```json
"statusBar.words": "{{count}} Wörter"   ✅ correct
"statusBar.words": "{{anzahl}} Wörter"  ❌ wrong — placeholder renamed
```

## Testing your translation

1. Change the language in **Settings → Appearance → Language** to your new locale
2. Restart the app
3. Browse through the UI to check all strings look right

## Installing a custom translation (without rebuilding)

If you want to test or use a translation that isn't shipped with Meridian yet:

1. Open **Settings → About → Open locales folder**
2. Drop your `XX.json` file into that folder
3. Restart Meridian
4. Your language will appear in **Settings → Appearance → Language**
```

const [availableLangs, setAvailableLangs] = useState<{ value: string; label: string }[]>([

])

useEffect(() => {
  const langNames: Record<string, string> = {
    en: 'English', ru: 'Русский', de: 'Deutsch', fr: 'Français',
    es: 'Español', ja: '日本語', zh: '中文', pt: 'Português',
    it: 'Italiano', nl: 'Nederlands', pl: 'Polski', ko: '한국어',

  ;(window.vault as any).listUserLocales?.().then((files: { lang: string }[]) => {
    const userLangs = files.map(({ lang }) => ({
      value: lang,
      label: langNames[lang] ?? lang.toUpperCase()
    }))
    setAvailableLangs((prev) => {
      const existing = new Set(prev.map(l => l.value))
      return [...prev, ...userLangs.filter(l => !existing.has(l.value))]
    })
  }).catch(() => {})
